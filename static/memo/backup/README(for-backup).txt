▼てがろぐ backup サブフォルダの説明
――――――――――――――――――
投稿本文の自動バックアップ機能を使いたい場合は、このサブフォルダが必要です。

その際、このサブフォルダには「書き込み権限」の付与が必要です。
●一般のウェブサーバでは、ディレクトリのパーミッションを 766 や 706 に設定します。
●SuEXECが使われているウェブサーバでは、パーミッションは 705 などのままで使えます。(※「さくらインターネット」や「ロリポップ」のサーバなどはこちら。)

自動バックアップ機能は、標準でONに設定されています。
ディレクトリへの書き込みが正常に可能なら、自動的にバックアップファイルが蓄積されていきます。
バックアップファイルは、標準設定では30日まで保存され、順次古いものから自動削除されます。

自動バックアップの設定は、CGIの管理画面から変更可能です。
自動バックアップのディレクトリ名自体は、CGIソース冒頭付近で設定(変更)可能です。

自動バックアップ機能の説明については、公式サイトの「自動バックアップ機能」項目もご参照下さい。
https://www.nishishi.com/cgi/tegalog/usage/#howtouse-autobackup
